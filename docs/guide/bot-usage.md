# Bot Discord 使用說明

Lomall Bot 提供完整的 Discord 工單管理功能，以下為所有指令的詳細使用方式。

---

## 快速開始

### 1. 初次設定

在 Discord 伺服器中輸入：

```
/lomall setup
```

這是使用 Lomall 的第一步，系統會將你的伺服器註冊到資料庫。

**權限需求：** 需要「管理伺服器」權限

**成功後會顯示：**
- ✅ 設定完成訊息
- 儀表板連結
- 快速使用提示

---

### 2. 建立工單面板

在你想讓使用者建立工單的頻道中輸入：

```
/ticket panel
```

這會發送一個包含「🎫 Create Ticket」按鈕的漂亮面板。使用者點擊按鈕即可建立工單。

---

### 3. 使用者建立工單（兩種方式）

#### 方式一：透過按鈕（推薦）

點擊「🎫 Create Ticket」按鈕後，會彈出一個表單：

| 欄位 | 必填 | 說明 |
|------|------|------|
| Subject | 是 | 問題簡短標題（最多100字） |
| Category | 否 | 分類：general / billing / technical / report / other |
| Description | 否 | 問題詳細描述（最多1000字） |

#### 方式二：透過指令

```
/ticket create subject:我的帳號有問題 category:technical
```

| 選項 | 必填 | 說明 |
|------|------|------|
| `subject` | 是 | 工單標題 |
| `category` | 否 | 分類（可選：General / Billing / Technical / Report / Other） |

---

### 4. 工單建立後會發生什麼？

1. **私密頻道** — 系統會自動建立一個名為 `ticket-主題-代碼` 的私密頻道
2. **權限隔離** — 只有你和管理團隊可以看到這個頻道
3. **歡迎訊息** — 頻道內會顯示工單資訊與「🔒 Close Ticket」關閉按鈕
4. **儀表板同步** — 工單會立即出現在網頁儀表板上

---

### 5. 管理團隊操作

#### 回覆工單

直接在 Discord 工單頻道中輸入訊息即可，訊息會自動同步到：
- Discord 工單頻道
- 網頁儀表板對話介面

#### 關閉工單

點擊工單頻道中的「🔒 Close Ticket」按鈕，或使用儀表板關閉功能。

關閉流程：
1. 點擊關閉按鈕
2. 系統倒數 5 秒
3. 工單標記為已關閉
4. 頻道在 5 秒後自動刪除

#### 重新開啟工單

在網頁儀表板中，找到已關閉的工單，點擊「Reopen」按鈕即可重新開啟。

---

### 6. 儀表板存取

```
/lomall dashboard
```

這會回傳一個包含儀表板連結的嵌入訊息。點擊連結後使用 Discord 帳號登入即可。

---

### 7. 伺服器設定

```
/lomall config
```

顯示目前伺服器的 Lomall 設定。

#### 修改設定

```
/lomall config setting:sla value:24
```

| 設定 | 類型 | 預設值 | 說明 |
|------|------|--------|------|
| `sla` | 數字 | 24 | SLA 回應時間（小時） |
| `autoClose` | 開啟/關閉 | 開啟 | 自動關閉閒置工單 |
| `autoCloseHours` | 數字 | 48 | 閒置多少小時後自動關閉 |

## 指令一覽

| 指令 | 說明 | 誰可以使用 |
|------|------|-----------|
| `/lomall setup` | 初次設定伺服器 | 管理員 |
| `/lomall dashboard` | 取得儀表板連結 | 所有人 |
| `/lomall config` | 檢視/修改設定 | 管理員 |
| `/ticket create` | 建立工單 | 所有人 |
| `/ticket panel` | 發送工單建立面板 | 管理員 |

---

## 權限說明

| Discord 角色 | 儀表板權限 | 可進行的操作 |
|-------------|-----------|-------------|
| Admin 角色 | 管理員 | 完整存取：設定、指派、檢視全部 |
| Support 角色 | 客服 | 管理工單、回覆對話 |
| Viewer 角色 | 檢視者 | 唯讀檢視工單與審計日誌 |
| 一般成員 | 無 | 僅能建立工單與在自己的工單頻道中發言 |
