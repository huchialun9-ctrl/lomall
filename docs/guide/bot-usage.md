## 1. Bot 概述

### 1.1 Lomall Bot 能做什麼？

| 功能 | 說明 |
|------|------|
| 🎫 工單建立 | 透過按鈕或指令建立支援工單 |
| 🔒 私密頻道 | 自動建立僅限當事人與團隊可見的頻道 |
| 💬 訊息同步 | Discord 與網頁儀表板雙向即時同步 |
| 🔘 按鈕操作 | 一鍵建立工單、一鍵關閉工單 |
| ⚙️ 伺服器設定 | 管理 SLA、自動關閉等參數 |
| 📊 儀表板連結 | 快速存取網頁管理介面 |

### 1.2 技術規格

| 項目 | 內容 |
|------|------|
| Discord.js 版本 | v14 |
| 指令類型 | Slash Commands（斜線指令） |
| 互動元件 | Button（按鈕）、Modal（表單） |
| 權限系統 | Discord 角色整合 |
| 後端通訊 | REST API + Socket.io 即時推播 |

## 2. 安裝與初次設定

### 2.1 邀請 Bot 至伺服器

在 [Discord Developer Portal](https://discord.com/developers/applications) 產生邀請連結，需勾選以下權限：

**必要權限：**
- `Send Messages` — 發送訊息
- `Manage Channels` — 建立與管理工單頻道
- `Manage Roles` — 設定頻道權限
- `Read Message History` — 讀取訊息記錄
- `Use Slash Commands` — 使用斜線指令

**建議權限：**
- `Administrator` — 簡化權限管理（可選）

### 2.2 初次設定指令

```
/lomall setup
```

這是**使用 Lomall 的第一步**，必須先執行此指令才能啟用所有功能。

**執行後系統會：**
1. 將 Discord 伺服器註冊到 Lomall 資料庫
2. 建立預設設定（SLA 24h、自動關閉啟用）
3. 回傳設定成功的確認訊息與儀表板連結

**注意事項：**
- 每個伺服器只需執行一次
- 需要「管理伺服器」（Administrator）權限
- 如果未執行此指令，其他指令會顯示「Guild not registered」錯誤

### 2.3 建立工單分類頻道（建議）

建議在 Discord 中建立一個名為 `Tickets` 的**頻道分類**，所有工單頻道會自動歸類到此分類下：

1. 在 Discord 頻道列表上方按右鍵 → 「建立分類」
2. 命名為 `Tickets`
3. Lomall Bot 會自動偵測此分類並將工單頻道建立於其下

如果沒有建立 `Tickets` 分類，工單頻道仍會正常建立，只是不會歸類到特定分類中。

## 3. 完整指令參考

### 3.1 `/lomall setup` — 初始化伺服器

**用途：** 初次設定 Lomall 系統

**語法：**
```
/lomall setup
```

**權限需求：** `管理伺服器`

**回應範例：**
```
✅ Lomall Setup Complete
Your server is now ready for ticket management.

Dashboard: [Open Dashboard](http://localhost:3000/login)
Quick Start: Use `/ticket panel` to create a ticket button
```

**錯誤處理：**
| 錯誤訊息 | 原因 | 解決方式 |
|----------|------|---------|
| `Guild not registered` | API 無法連線 | 確認 API 服務已啟動 |
| `403 Forbidden` | Bot 權限不足 | 檢查 Bot 是否有管理頻道權限 |
| `Request failed` | 網路問題 | 確認 API 伺服器正在運行 |

---

### 3.2 `/lomall dashboard` — 取得儀表板連結

**用途：** 取得 Lomall 網頁管理介面的登入連結

**語法：**
```
/lomall dashboard
```

**回應範例（Ephemeral，僅自己可見）：**
```
🌐 Lomall Dashboard

Access your dashboard to manage tickets.

[Open Dashboard](http://localhost:3000/login)

Sign in with Discord to continue
```

**使用提示：**
- 連結僅對該 Discord 帳號有效
- 需使用 Discord OAuth2 登入
- 儀表板權限取決於使用者在 Discord 中的角色

---

### 3.3 `/lomall config` — 檢視與修改設定

**用途：** 檢視或修改伺服器的 Lomall 設定參數

#### 檢視目前設定

```
/lomall config
```

**回應範例：**
```
⚙️ 伺服器名稱 Configuration

SLA (hours): `24`
Auto-close: `Enabled`
Auto-close (hours): `48`

Use /lomall config <setting> <value> to update
```

#### 修改單一設定

```
/lomall config setting:sla value:24
```

**可用設定參數：**

| 設定名稱 | 類型 | 預設值 | 說明 | 範例值 |
|----------|------|--------|------|--------|
| `sla` | 數字（小時） | `24` | SLA 目標回應時間 | `12`, `24`, `48` |
| `autoClose` | 布林值 | `true` | 是否自動關閉閒置工單 | `true`, `false` |
| `autoCloseHours` | 數字（小時） | `48` | 閒置多久後自動關閉 | `24`, `72`, `168` |

**使用範例：**

```bash
# 將 SLA 改為 12 小時
/lomall config setting:sla value:12

# 關閉自動關閉功能
/lomall config setting:autoClose value:false

# 將自動關閉時間改為 72 小時
/lomall config setting:autoCloseHours value:72
```

**補充說明：**
- `autoClose` 接受的值：`true`、`false`、`yes`、`no`
- 修改後立即生效
- 設定會儲存至資料庫，重啟 Bot 不會遺失

---

### 3.4 `/ticket create` — 建立工單（指令方式）

**用途：** 透過 Slash Command 直接建立工單

**語法：**
```
/ticket create subject:<標題> category:<分類>
```

**參數說明：**

| 參數 | 必填 | 限制 | 說明 |
|------|------|------|------|
| `subject` | ✅ 是 | 1-100 字元 | 工單的簡短標題 |
| `category` | ❌ 否 | 選項清單 | 工單分類，預設為 `general` |

**可選分類：**

| 選項 | 值 | 適用情況 |
|------|-----|---------|
| General | `general` | 一般問題 |
| Billing | `billing` | 帳單或付款問題 |
| Technical | `technical` | 技術支援 |
| Report | `report` | 檢舉使用者或問題回報 |
| Other | `other` | 其他 |

**使用範例：**

```bash
# 基本用法（僅標題）
/ticket create subject:無法登入我的帳號

# 指定分類
/ticket create subject:付款發票問題 category:billing

# 技術問題回報
/ticket create subject:伺服器連線逾時 category:technical
```

**回應範例：**
```
✅ Ticket created! Check #ticket-無法登入-a1b2c3
```

### 3.5 `/ticket panel` — 發送工單建立面板

**用途：** 在指定頻道中發送一個含「🎫 Create Ticket」按鈕的互動面板

**語法：**
```
/ticket panel
```

**權限需求：** 管理員

**面板內容：**
- 標題：🎫 Create a Ticket
- 說明：引導使用者建立工單
- 分類列表：顯示所有可選分類
- 預估回應時間
- 按鈕：🎫 Create Ticket

**建議使用方式：**
- 在 `#help` 或 `#support` 等公開頻道中使用
- 每個頻道可以發送多個面板
- 面板可以透過 Discord 的訊息管理功能釘選

**注意事項：**
- 按下按鈕後會彈出 Modal 表單（詳見 [第 5 節](#5-modal-表單系統)）
- 面板本身不會過期，按鈕永遠有效

## 4. 按鈕互動元件

Lomall Bot 使用 Discord Button 元件提供直覺式操作。

### 4.1 🎫 Create Ticket（建立工單）

| 屬性 | 內容 |
|------|------|
| 樣式 | Primary（藍色） |
| 位置 | `/ticket panel` 發送的面板中 |
| 功能 | 開啟工單建立表單（Modal） |

**行為說明：**
1. 點擊後跳出 Modal 表單
2. 填寫必要欄位後送出
3. 系統建立工單與私密頻道
4. 使用者收到成功通知

### 4.2 🔒 Close Ticket（關閉工單）

| 屬性 | 內容 |
|------|------|
| 樣式 | Danger（紅色） |
| 位置 | 每個工單頻道的歡迎訊息中 |
| 功能 | 關閉並清理工單頻道 |

**行為說明：**
1. 點擊關閉按鈕
2. 系統發送「🔒 Closing ticket in 5 seconds...」
3. 等待 5 秒（讓使用者有機會取消）
4. 系統透過 API 將工單標記為 `closed`
5. 發送「✅ Ticket closed. Channel will be deleted shortly.」
6. 再等待 5 秒
7. 自動刪除 Discord 頻道

**如果發生錯誤：**
- 如果 API 無法連線，按鈕會顯示錯誤訊息
- 頻道刪除失敗時會留下紀錄，需手動刪除

## 5. Modal 表單系統

Lomall 使用 Discord Modal（彈出式表單）收集工單資訊，在使用者點擊「🎫 Create Ticket」按鈕時觸發。

### 5.1 表單欄位說明

| 欄位 | ID | 必填 | 樣式 | 最大長度 | 說明 |
|------|-----|------|------|---------|------|
| 主旨 | `subject` | ✅ 是 | Short（單行） | 100 字 | 問題簡短標題 |
| 分類 | `category` | ❌ 否 | Short（單行） | 20 字 | 輸入分類名稱 |
| 描述 | `description` | ❌ 否 | Paragraph（多行） | 1000 字 | 問題詳細描述 |

### 5.2 分類輸入說明

`category` 欄位接受以下值（不分大小寫）：

| 輸入值 | 對應分類 |
|--------|---------|
| `general` 或留空 | general |
| `billing` | billing |
| `technical` | technical |
| `report` | report |
| `other` | other |

無效的分類值會自動回退為 `general`。

### 5.3 送出後的處理流程

```
使用者點擊 Create Ticket 按鈕
        │
        ▼
  彈出 Modal 表單
        │
        ▼
  使用者填寫並送出
        │
        ▼
  ┌─────────────────┐
  │ 驗證輸入內容      │── 無效 → 顯示錯誤
  └─────────┬───────┘
            │ 有效
            ▼
  ┌─────────────────┐
  │ 透過 API 建立工單 │── 失敗 → 顯示錯誤訊息
  └─────────┬───────┘
            │ 成功
            ▼
  ┌─────────────────┐
  │ 建立 Discord 頻道│
  │ 設定權限        │
  │ 發送歡迎訊息    │
  └─────────────────┘
            │
            ▼
  顯示 ✅ Ticket created! 通知
```

## 6. 工單生命週期

### 6.1 狀態流程圖

```
        [建立工單]
            │
            ▼
     ┌─────────────┐
     │   Open      │ ◄──── 客服與使用者來回溝通
     │  (開啟中)   │
     └──────┬──────┘
            │
     ┌──────▼──────┐
     │  Resolved   │     問題已解決，等待確認
     │  (已解決)   │
     └──────┬──────┘
            │
     ┌──────▼──────┐
     │   Closed    │     工單關閉，頻道刪除
     │  (已關閉)   │
     └──────┬──────┘
            │
            │ (可透過儀表板重新開啟)
            ▼
     ┌─────────────┐
     │   Open      │
     └─────────────┘
```

### 6.2 各階段詳細說明

#### 階段一：Open（開啟中）

| 項目 | 內容 |
|------|------|
| 觸發方式 | `/ticket create` 或按鈕 → Modal |
| Discord 狀態 | 私密頻道存在，所有參與者可發言 |
| 儀表板狀態 | 顯示在「Open」分頁 |
| 允許操作 | 回覆訊息、指派人員、關閉工單 |

#### 階段二：Resolved（已解決）

| 項目 | 內容 |
|------|------|
| 觸發方式 | 透過儀表板設定 |
| Discord 狀態 | 頻道仍存在，可繼續對話 |
| 儀表板狀態 | 顯示在「Resolved」分頁 |
| 允許操作 | 重新開啟、關閉、回覆 |

#### 階段三：Closed（已關閉）

| 項目 | 內容 |
|------|------|
| 觸發方式 | 點擊 🔒 Close Ticket 按鈕或儀表板關閉 |
| Discord 狀態 | 頻道已刪除（但有轉存紀錄） |
| 儀表板狀態 | 顯示在「Closed」分頁 |
| 允許操作 | 透過儀表板重新開啟 |

### 6.3 自動關閉機制

如果啟用了自動關閉（`autoClose: true`），系統會：

1. 監控所有 Open 狀態的工單
2. 計算最後活動時間
3. 如果超過 `autoCloseHours`（預設 48 小時）無人回覆
4. 自動將工單標記為 Closed

### 6.4 轉存系統

工單關閉時會自動產生 HTML 封存檔（Transcript），包含：
- 完整對話紀錄
- 參與者資訊
- 時間戳記
- 工單元資料（分類、優先級等）

轉存檔會永久儲存於資料庫中，可透過儀表板查詢。

## 7. 管理團隊操作指南

### 7.1 回覆工單

直接在 Discord 工單頻道中輸入訊息即可。

**自動同步：**
```
Discord 訊息 ────→ API ────→ 儀表板即時顯示
```

**回覆辨識：**
- 系統會自動判斷回覆者是否為管理團隊成員
- 判斷依據：使用者的 Discord 角色是否包含 `admin`、`support` 或 `staff` 關鍵字
- 管理者的回覆在儀表板上會有「Staff」標記

### 7.2 關閉工單

**方式一：點擊關閉按鈕**

在工單頻道的歡迎訊息中點擊「🔒 Close Ticket」按鈕。

**方式二：從儀表板關閉**

1. 登入 Lomall 儀表板
2. 找到要關閉的工單
3. 點擊「Close」按鈕

### 7.3 重新開啟工單

已關閉的工單可以重新開啟，但僅能透過**網頁儀表板**操作：

1. 登入儀表板
2. 切換到「Closed」分頁
3. 點擊目標工單
4. 點擊「Reopen」按鈕

注意：重新開啟後，原本的 Discord 頻道已刪除，需要手動重新建立新的對話頻道，或直接在儀表板中進行對話。

### 7.4 指派工單

可將工單指派給特定團隊成員（透過儀表板操作）：

1. 開啟工單詳情頁
2. 點擊「Assign」
3. 選擇要指派的管理員

指派後會在審計日誌中留下紀錄。

## 8. 訊息同步機制

### 8.1 同步方向

```
┌──────────┐          ┌──────────┐          ┌──────────┐
│ Discord  │ ──API──▶ │ Lomall   │ ◀──API── │ Web      │
│ Bot      │          │ Database │          │ Dashboard│
│ 頻道訊息  │ ◀─────── │          │ ────────▶ │ 對話介面  │
└──────────┘          └──────────┘          └──────────┘
```

### 8.2 同步內容

| 項目 | 同步方式 | 更新頻率 |
|------|---------|---------|
| 文字訊息 | REST API | 即時（送出後立即同步） |
| 工單狀態 | REST API + Socket.io | 即時 |
| 審計日誌 | REST API | 即時 |

### 8.3 哪些訊息會被同步

符合以下所有條件的訊息會被同步到 API：
1. 發送者在 Bot 快取中的工單頻道
2. 訊息內容為純文字
3. 不是 Bot 本身發送的訊息

### 8.4 隱私與安全

- 僅工單頻道內的訊息會被同步
- 一般頻道的對話不受影響
- 同步的訊息包含發送者 Discord ID，可追溯來源

## 9. 頻道管理與權限隔離

### 9.1 頻道命名規則

```
ticket-<主旨>-<工單ID前6碼>

範例：
ticket-無法登入-a1b2c3
ticket-付款問題-d4e5f6
```

主旨中的特殊字元會被取代為連字號 `-`，並限制在 20 個字元以內。

### 9.2 權限覆寫（Permission Overwrites）

新建立的工單頻道會自動設定以下權限：

| 角色/使用者 | 允許 | 拒絕 |
|------------|------|------|
| @everyone | — | 檢視頻道、發送訊息 |
| 工單建立者 | 檢視頻道、發送訊息、讀取歷史 | — |
| 管理員角色 | 檢視頻道、發送訊息、管理頻道 | — |

### 9.3 權限隔離示意圖

```
#一般頻道（所有人都看得到）
  #general
  #random
  #help ← 這裡放 /ticket panel 按鈕

#Tickets 分類（只有當事人看得到）
  ticket-問題-a1b2c3  ← 使用者A + 管理團隊
  ticket-問題-d4e5f6  ← 使用者B + 管理團隊
```

### 9.4 頻道清理

關閉工單後，系統會：
1. 等待 5 秒（緩衝時間）
2. 發送關閉確認訊息
3. 再等待 5 秒
4. 自動刪除 Discord 頻道

如果頻道刪除失敗（例如 Bot 權限被修改），管理員可以手動刪除。

## 10. 伺服器設定管理

### 10.1 設定項目總覽

| 設定鍵 | 型態 | 預設值 | 說明 | 修改方式 |
|--------|------|--------|------|---------|
| `sla` | number | `24` | SLA 回應時間（小時） | `/lomall config` 或儀表板 |
| `autoClose` | boolean | `true` | 是否啟用自動關閉 | `/lomall config` 或儀表板 |
| `autoCloseHours` | number | `48` | 閒置多少小時後自動關閉 | `/lomall config` 或儀表板 |
| `categoryId` | string | `null` | 工單頻道所屬的分類 ID | 自動偵測 |
| `roles` | object | `{}` | 角色與權限對應表 | 儀表板 |

### 10.2 設定優先級

```

```

所有設定會即時生效並儲存於資料庫中，重新啟動 Bot 不會遺失。

### 10.3 未執行 `/lomall setup` 的影響

| 功能 | 影響 |

## 13. 常見問題與故障排除

### 13.1 指令執行問題

**Q: 輸入指令後沒有任何回應？**
- 確認 Bot 是否在線上（Discord 成員列表中查看）
- 確認 Bot 有「讀取訊息」和「發送訊息」權限
- 重新邀請 Bot 到伺服器

**Q: 顯示「This command can only be used in a server」**
- 這個指令只能在 Discord 伺服器中使用，不能在私人訊息中使用

**Q: 顯示「Guild not registered」**
- 尚未執行 `/lomall setup`
- 或 API 伺服器未啟動
- 解決方式：執行 `/lomall setup` 或啟動 API

**Q: Slash Command 沒有出現在選單中**
- 等待幾秒後重新輸入 `/`
- 如果持續未出現，請重新邀請 Bot（需有 `applications.commands` 權限範圍）

### 13.2 工單建立問題

**Q: 工單建立失敗，顯示 API 錯誤**
- 確認 API 伺服器正在運行
- 確認 PostgreSQL 資料庫已連線
- 檢查 `.env` 設定是否正確

**Q: 按鈕點擊後沒有反應**
- Bot 可能尚未處理完前一個請求
- 重新點擊一次
- 如果持續無反應，重新啟動 Bot

**Q: Modal 表單無法送出**
- 檢查必填欄位是否已填寫
- 檢查文字長度是否超過限制（Subject 100 字、Description 1000 字）
- Discord Modal 有 15 分鐘的逾時限制

**Q: 私密頻道沒有自動建立**
- 檢查 Bot 是否有「管理頻道」權限
- 檢查伺服器頻道數量上限是否已滿
- 手動檢查 API 日誌是否有錯誤

### 13.3 權限問題

**Q: 使用者看不到自己的工單頻道**
- 確認 Bot 在建立頻道時有設定 Permission Overwrites
- 檢查頻道權限設定中是否有該使用者的存取權限
- 重新邀請 Bot 並確保有「管理角色」權限

**Q: 管理員看不到所有工單頻道**
- 確認管理員的 Discord 角色有包含 `admin` 關鍵字
- 或是手動將管理員加入頻道權限

### 13.4 訊息同步問題

**Q: Discord 訊息沒有出現在儀表板**
- 確認該頻道是 Lomall 建立的工單頻道
- 確認 Bot 有啟動並在線上
- 檢查 API 日誌是否有錯誤訊息
- 嘗試在頻道中再發送一次訊息

### 13.5 其他問題

**Q: 關閉工單後頻道沒有被刪除**
- Bot 可能在關閉過程中斷線
- 可以手動刪除頻道
- 檢查 Bot 是否有「管理頻道」權限

**Q: Bot 突然離線**
- 檢查 Bot Token 是否有效
- 重新啟動 Bot 程式
- 檢查伺服器資源是否充足

**Q: 如何重置所有設定？**
- 重新執行 `/lomall setup` 會更新伺服器資訊
- 如需完全重置，需從資料庫中刪除該伺服器的紀錄

## 14. 限制與注意事項

### 14.1 Discord API 限制

| 項目 | 限制 |
|------|------|
| Slash Command 名稱長度 | 1-32 字元 |
| Modal 欄位數量上限 | 5 個 |
| Modal 逾時時間 | 15 分鐘 |
| 頻道名稱長度 | 2-100 字元 |
| 伺服器頻道數量上限 | 500 個 |
| 嵌入訊息欄位上限 | 25 個 |

### 14.2 Lomall 系統限制

| 項目 | 限制 |
|------|------|
| Subject 最大長度 | 100 字元 |
| Description 最大長度 | 1000 字元 |
| 工單頻道名稱長度 | 約 30 字元（含前綴與 ID） |
| 訊息同步 | 僅同步文字內容（不含附件、貼圖） |

### 14.3 注意事項

1. **Bot 權限不可撤銷**：一旦開始使用，請勿移除 Bot 的管理頻道權限，否則工單功能會失效
2. **Token 安全**：Bot Token 應妥善保管，若遭外洩應立即在 Discord Developer Portal 重新產生
3. **API 依賴**：Bot 的所有功能依賴 API 伺服器，API 離線時 Bot 僅能回應基本指令
4. **頻道清理**：關閉工單後頻道會自動刪除，如有需要請在關閉前備份重要資訊
5. **資料庫備份**：建議定期備份 PostgreSQL 資料庫，以防資料遺失

## 15. 最佳實踐建議

### 15.1 伺服器設定建議

| 場景 | 建議設定 |
|------|---------|
| 小型社群（< 100 人） | SLA: 24h, Auto-close: 48h |
| 大型社群（> 1000 人） | SLA: 12h, Auto-close: 24h |
| 付費客戶支援 | SLA: 4h, Auto-close: 72h |
| 開發測試階段 | Auto-close: disabled |

### 15.2 頻道規劃

```
# 建議的頻道結構

📋 INFO
  #rules              ← 伺服器規則
  #announcements      ← 公告

❓ SUPPORT
  #help               ← 放置 /ticket panel 面板
  #faq                ← 常見問題（非必要）

📁 TICKETS（分類）
  ticket-xxx-xxxxxx   ← 自動建立的工單頻道

🔒 STAFF ONLY
  #staff-chat         ← 管理團隊內部討論
  #ticket-logs        ← 工單活動紀錄（可選）
```

### 15.3 管理團隊 SOP

**建議的工單處理流程：**

1. **即時回應**：管理員收到工單建立通知後，應在 SLA 時間內回應
2. **分類處理**：根據 category 分配給對應的負責人
3. **定期檢查**：每天檢查 Open 工單，避免遺漏
4. **關閉確認**：關閉前確認使用者問題已解決
5. **紀錄保存**：重要工單可從儀表板匯出轉存紀錄

### 15.4 避免常見錯誤

- ❌ 不要在工單頻道中討論不相關的話題
- ❌ 不要手動刪除工單頻道（應使用關閉功能）
- ✅ 定期檢查 `/lomall config` 設定是否正確
- ✅ 確保至少有一位管理員有 Admin 權限
- ✅ 在公開頻道中說明工單系統的使用方式
